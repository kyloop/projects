Classification Test Set #5¶

Project task: Binary Classification Challenge¶

The primary purpose of this project is to make Binary (i.e. 1 or 0) prediction on test set of data with 10k records. The training and Testing set of data include 40k and 10k of records with 100 features respectively. The classifiers for this project are XGB and Random Forest Classifiers. The scope of project includes EDA, Visualization, Modeling and Model determination. The main metrics to measure the performance of the Models are AUC and Recall scores for the highly biased dataset.

Check data type of columns which are NOT in "float" format¶

The following step is to check the features which are not in "float" format are taken action to

  • 1) De-categorize features (e.g. Convert to float value) for modeling purpose.
  • 2) Dummy-out the Categorical features in the binary format (i.e. 1 or 0)

Features:

  • x34: Dummy-out x34 feature into Ten binary feature (i.e. 1 or 0)
  • x35: Convert x35 feature into Ordinal feature (i.e. Monday -> 1, Tue -> 2 etc)
  • x41: Extract the digits including +/- signs from the original string format (e.g. $-865.28 -> 865.28)
  • x45: Extract the digits including +/- signs from the original string format (e.g. 0.02% -> 0.02)
  • x68: Convert x68 feature into Ordinal feature (i.e. Jan -> 1, Feb -> 2 etc)
  • x93: Convert x93 feature into proportion values based on their Categorical value
    • asia -> 0.8846, america -> 0.079175, europe -> 0.03605

Display Dataset Property and Statistics¶

This dataset contains 100 features with with 40000 instances.

  • The following shows some information about the features¶
  • non_empty_counts: Number of Non-empty Cells of the specific feature
  • nulls: Number of Null cells of the specific feature
  • pct_nulls: Percent of null cells of the specific feature
  • dtype: Type of the feature
  • mean: Averge Value of the values of the specific feature
  • median: median value of the values of the specific feature
  • std: standard deviation of the values of the specific feature
  • skewness: skewness of the values of the specific feature
  • corr_x_y: Correlation value between the feature and Target Value

Summary:

  • nulls: Missing Values taken very least portion among all features in the dataset, so that records are not going to be eliminated. Instead, Missing values are going to be replaced by Mean values for feature with "float64" format and Mode values for feature with "uint8" format.
  • dtypes: Types of features are mostly in "float64" format, and then target variable is in "int8" and the rest features are in "int8" and "uint8" format.
  • skewness: The values of skewness of the features are ALL between -3 and 3, so that missing values are going to be replace by their Mean values or Mode Values
  • correlation: As shown below, the highest correlation values are around 0.2 (i.e. x97, x75). we might need to utilize the features with higher correlation values with the Target variables to build the model depending upon the performance of the models.

from scipy.stats import skew
def DisplayT(DF, target_feature):
    counts=DF.count()
    nulls=DF.apply(lambda col: col.isnull().sum())
    pct_nulls = np.round(nulls / DF.shape[0],5)
    means = DF.mean()
    medians = DF.median()
    sds = DF.std()
    
    skewness = DF.apply(lambda x: skew(x[x.isnull()==False]))
    
#    uniques=DF.apply(lambda col: [col.unique()])
#    count_uniques = DF.apply(lambda col: len([col.unique()]))
#    count_uniques=[len(lst) for lst in uniques.iloc[0,:].values]

    dtypes=DF.dtypes
    #tbl=pd.DataFrame(list(zip(uniques.iloc[0,:].values)), columns=["uniqueness"])
    
    tbl=pd.DataFrame(list(zip(DF.columns,
                              counts, 
                              nulls,
                              pct_nulls*100 ,
                              dtypes,
                              means,
                              medians,
                              sds,
                              skewness,
                              #uniques.iloc[0,:].values,count_uniques,  
                              DF.corr()[target_feature])), 
                     columns=["feature","non_empty_counts","nulls","pct_nulls","dtypes", 
                              "means",
                              "medians",
                              "sds",
                              "skewness",
                              #"uniqueness", "count_unique",
                              "corr_x_y" ])
    tbl=tbl.set_index("feature")
    tbl=tbl[tbl.index != target_feature]
    return(tbl)

def gridXGBCV(clf,X, y, cvFolds, gridParas, scoring):
    gridcv = GridSearchCV(clf, 
                          param_grid= gridParas, 
                          scoring=scoring, 
                          verbose=10, 
                          cv=cvFolds,
                          n_jobs =-1)
    gridcv.fit(X,y)
    result=gridcv.cv_results_

Perform t-test to determine the best model¶

This part is to perform t-test in order to determine the best models from the top two models. The metric we utilize in this part is Recall because we would like to focus small portion Target Class "1"

We first need to set up the Hypothesis as shown below:

  • Null: model with original features (Model 1) = model with subset of features (Model 3)
  • Alternate: model with original features (Model 1) <> model with subset of features (Model 3)

Since P-value < t (i.e. 1.57699e-31 < 14.0761), so that we could reject the Null Hypothesis which assumes both models perform equally well. In other words, The model built with Original Dataset (Model 1) and model built with subset of dataset (Model 3) should perform differently . According to the 100 Recall Resampling experiment, the average Recall from model with original dataset (i.e. 89.5%) is superior than model with subset of data (i.e. 85.5%).
